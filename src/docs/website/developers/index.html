 These pages are for people interested in using the development copy of PETSc or contributing to the PETSc libraries. It is intended only for people who are experienced with building, using and debugging PETSc. If you cannot use makefiles, a debugger and EMACS etags then please don't access these pages.

Obtaining PETSc:

To obtain the development version of PETSc you have four options
  1.  download and install the free BitKeeper download client
  2.  access the nightly tarball snapshot at ftp://info.mcs.anl.gov/pub/petsc/.petsc-dev.tar.gz
  3. license BitKeeper (inexpensive academic licences available).
  4. [experimental] obtain PETSc repostiory using darcs
 All three create a local directory with the name petsc-dev.

1) To use the free BitKeeper client Install the free client from here http://www.bitkeeper.com/Hosted.Downloading.html. Make sure the directory where you have compiled this code is in your path.
  • sfioball bk://petsc.bkbits.net/petsc-dev petsc-dev
  • cd petsc-dev/python
  • sfioball bk://sidl.bkbits.net/BuildSystem BuildSystem
To update PETSc
  • cd petsc-dev
  • update bk://petsc.bkbits.net/petsc-dev
  • cd python/BuildSystem
  • update bk://sidl.bkbits.net/BuildSystem

2) To use the tarball snapshot, simply download ftp://info.mcs.anl.gov/pub/petsc/.petsc-dev.tar.gz and ungzip and untar it. To update PETSc simply get a new copy of the tar file.

3) To use the licensed BitKeeper, invoke the following command to obtain PETSc.
  • bk  clone  bk://petsc.bkbits.net/petsc-dev
  • cd petsc-dev/python
  • bk clone bk://sidl.bkbits.net/BuildSystem
For additional help on using BitKeeper you can use
  • bk helptool
To update PETSc
  • cd petsc-dev
  •  bk pull
  • cd python/BuildSystem
  • bk pull
To modify your local copy of PETSc, you can use bk edit and bk citool, as described in bk helptool. These changes can be shared with the PETSc community by pushing to the BkBits repository. You will need to send an SSH 1 public key to petsc-maint along with a description of the changes. Documentation for the development version (updated nightly).

4) Using Darcs obtain the PETSc repository snapshot. We export our BitKeeper repository into darcs a couple of times a day.
  • darcs get --partial http://www-unix.mcs.anl.gov/petsc/petsc-as/snapshots/petsc-dev-darcs petsc-dev
  • cd petsc-dev/python
  • darcs get --partial http://www-unix.mcs.anl.gov/petsc/petsc-as/snapshots/BuildSystem-darcs BuildSystem
To update PETSc
  • cd petsc-dev
  • darcs pull
  • cd python/BuildSystem
  • hdarcs pull

Note: this is highly experimental and useful only if a PETSc snapshot with revision history is necessary [and you don't have access to BitKeeper]

Compiling and Using the Development Copy:

Use config/configure.py to install PETSc, (trouble, please contact us petsc-maint@mcs.anl.gov).

Subscribe to the mailing list. Send mail to majordomo@mcs.anl.gov with the message: subscribe petsc-dev

Building Documentation:

The documentation tools are now automatically downloaded and installed by config/configure.py. However, as a failsafe, we provide them here as well.

The following additional tools are required to build the PETSc documentation:

  • Sowing:  A text processing tools developed by Bill Gropp

  • C2html: A text processing package required to generate the PETSc documentation.

  • A version of pdflatex (for example in teTeX):This package might already be installed on most systems. It is required to generate the users manual (part of the PETSc documentation)

   Once pdflatex (from teTeX) is in your PATH you can build the documentation with:

  • make alldoc LOC=${PETSC_DIR}
BuildSystem:

PETSc configure relies on BuildSystem package which is in a different BitKeeper repository bk://sidl.bkbits.net/BuildSystem. This package is automatically downloaded (into petsc-dev/python) the first time config/configure.py is invoked.
