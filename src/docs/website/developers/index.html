 These pages are for people interested in developing and contributing to the PETSc libraries. It is intended only for people who are experienced with building, using and debugging PETSc. If you cannot use makefiles, a debugger and EMACS etags then please don't access these pages.

Obtaining PETSc:

To obtain the development version of PETSc, download and install BitKeeper (free) from bitkeeper.com. Using BitKeeper, invoke the following command to obtain PETSc. (This creates a local directory with the name petsc-dev)
  • bk  clone  http://petsc.bkbits.net/petsc-dev  petsc-dev
For additional help on using BitKeeper you can use
  • bk helptool
We strongly recomment that all users of the development version of PETSc use the automatic configuration to install PETSc, rather than editing the bmake files manually (if you have trouble with the automatic configuration, please do contact us for help). If you must use the manual configuration do make allfortranstubs etags from the PETSc directory just before doing make all.


Updating PETSc:

The next time you need to update your copy of PETSc with the changes we made (since your last update), you can do the following:

  • make update(from inside the petsc-dev directory tree)

Building Documentation:

The documentation tools are now automatically downloaded and installed by configure. However, as a failsafe, we provide them here as well.

The following additional tools are required to build the PETSc documentation:

  • Sowing:  A text processing tools developed by Bill Gropp. This is automatically downloaded and installed by configure

  • C2html: A text processing package required to generate the PETSc documentation. This is automatically downloaded and installed by configure

  • A version of pdflatex (for example in teTeX):This package might already be installed on most systems. It is required to generate the users manual (part of the PETSc documentation)

   Once pdflatex (from teTeX) is in your PATH you can build the documentation with:

  • make alldoc LOC=${PETSC_DIR}.
