    These pages are for people interested in developing and contributing to the PETSc libraries. It is intended only for people who are experienced with building, using and debugging PETSc. We cannot provide support (debugging, building, etc) for people using the development copy of PETSc. If you cannot use a debugger and EMACS etags then please don't access these pages.

   To obtain development version of PETSc, download and install BitKeeper from bitkeeper.com. This is a free download, but registration might be required. This is a version control software similar to CVS. Using bitkeeper, invoke the following command to obtain PETSc. (This creates a local directory with the name petsc-dev)

  •    bk  clone  http://petsc.bkbits.net/petsc-dev   petsc-dev

  The next time you need to update your copy of PETSc with the changes we made (since your last update), you can do the following:

  • cd petsc-dev

  • bk pull

   However we recommend creating a new repository (using bk clone) - this way, you still have the old clone/libraries for use in case the latest snapshot is buggy. It is always possible that the current development snapshot is in an unuseable state (with errors when building libraries or examples randomly crashing)

  For additional help on using BitKeeper you can use

  • bk helptool

  Also, we recommend creating a new PETSC_ARCH - for example:  linux_local (i.e copy files from bmake/linux over to bmake/linux_local) and make edits suitable for your configuration in this copy - this way  you don't have to edit sources in BitKeeer.
 
The following additional tools are required to build the fortran interface and PETSc documentation:

  • Sowing:  A text processing tools developed by Bill Gropp. This package is primarily required to generate the fortran stubs.  It is  also used to generate some of the PETSc documentation.

  • C2html: A a text processing package required to generate the PETSc documentation.

  • teTeX: This package might already be installed on most systems. It is required to generate the users manual (part of the PETSc documentation)

   Once the above tools are available in the PATH, the following commands can be used to build the fortran stubs and the documentation (should be invoked in ${PETSC_DIR}):

  • make allfortranstubs : This builds the fortran stubs in src/fortran/auto dir

  • make alldoc allhtml LOC=${PETSC_DIR} : This builds all the documentation including the man pages, the user's manual, HTML formatted source files.
